(?<named>regexp+)

Named capture groups for Java 5/6 regular expressions

Download named-regexp-0.1.9.jar

about!

named-regexp is a thin wrapper for good ol' java.util.regex with support for named capture groups in Java 5/6. (yes, Java 7 already does this...)

The current version is 0.1.9.

snippet.

The construct for named groups is the same as in Java 7: (?<name>...), as shown below.

NamedPattern p = NamedPattern.compile("abc(?<foo>\\d+)xyz");
NamedMatcher m = p.matcher("abc123xyz");
System.out.println(m.find());       // true
System.out.println(m.group("foo")); // 123

back references

To back-reference a named group in a pattern, use \k<name>, as in: 
NamedPattern p = NamedPattern.compile("abc(?<foo>\\d+)def\\k<foo>");
NamedMatcher m1 = p.matcher("abc123def123ghi");
NamedMatcher m2 = p.matcher("abc123def456ghi");
System.out.println(m1.find());       // true
System.out.println(m1.group("foo")); // 123
System.out.println(m2.find());       // false

replacement references

To reference a named group for string replacement, use ${name}, as in: 
NamedPattern p = NamedPattern.compile("abc(?<foo>\\d+)def");
NamedMatcher m = p.matcher("abc123def");
System.out.println(m.replaceAll("${foo}456 ${foo}")); // 123456 123

download.

You can download named-regexp-0.1.9; or include it as a Maven dependency: 
<dependency>

  <artifactId>named-regexp</artifactId>
  <version>0.1.9</version>
</dependency>
